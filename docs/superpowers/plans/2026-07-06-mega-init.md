# mega init Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Strict TDD: failing test first тЖТ red тЖТ minimal impl тЖТ green тЖТ commit.

**Goal:** `mega init` collapses onboarding (hooks + mcp + workspace-saver-enable + `mega gui`) into one command тАФ first visible saving in minutes.

**Architecture:** A `runInit` orchestrator that calls the existing `runHooksInstall` / `runMcpInstall` / the workspace-saver-enable fn / `runGui`, with injected deps for testability. Default mode `balanced`; auto-run + summary; `--yes` silent; `--no-gui` skips the dashboard.

**Tech Stack:** TypeScript ESM, Citty, Vitest. Package: `apps/cli`.

**Spec:** `docs/superpowers/specs/2026-07-06-mega-init-design.md`. Risk MEDIUM тЖТ code-reviewer + critic.

**Anchors:** `apps/cli/src/commands/hooks/install.ts` `runHooksInstall(RunHooksInstallInput): 0|1`; `apps/cli/src/commands/mcp/install.ts` `runMcpInstall(RunMcpInstallInput): Promise<0|1>`; `apps/cli/src/commands/gui.ts` `runGui(RunGuiInput): Promise<RunGuiResult>`; the workspace-saver-enable fn under `apps/cli/src/commands/session/saver/` (grep for the `workspace enable` run fn тАФ wire the real export); `apps/cli/src/main.ts` subCommands (register `init` like `gui`/`trace`); store resolution `readStoreEnv`/`resolveStorePath`.
